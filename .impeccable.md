# AI Tutor — Design Context

## Design Context

### Users
Three distinct roles share the platform:
- **Students** — the primary audience; they learn through lessons, AI-generated content, quizzes, flashcards, and an AI chat tutor. They need clarity and motivation to stay on task.
- **Teachers / Instructors** — upload lesson materials, manage content, and monitor student progress. They need efficient, information-dense views.
- **Admins** — manage the platform holistically; need clear dashboard-style layouts with strong data hierarchy.

The interface must serve all three without feeling overloaded — achieved through strong information hierarchy and role-appropriate views rather than multiple visual languages.

### Brand Personality
**Motivated · Energetic · Focused**

The product should feel like a high-performance learning tool — purposeful and fast, not playful or casual. It gives students the confidence that they're studying *smart*, not just studying hard.

### Aesthetic Direction
- **Reference**: [NotebookLM](https://notebooklm.google.com/) — clean, structured, AI-first, information-dense. Strong sidebar + main content layout, purposeful use of color, professional but not sterile.
- **Anti-reference**: Nothing explicitly, but avoid: overly gamified UIs (Duolingo-excessive badges), cluttered dashboard overload, or generic SaaS blandness.
- **Theme**: Light mode default, full dark mode. Both must feel equally intentional — dark mode is not an afterthought.
- **Accent**: `#ff6900` (orange) — the established brand color. Use it purposefully: active states, CTAs, AI-related highlights. Never overuse it.
- **Active / highlight tint**: `#fff3e3` background + `#ff6900` text (light mode); `rgba(255,105,0,0.15)` bg + `#ff6900` text (dark mode).
- **Surfaces (light)**: `#ffffff` primary, `#f8f9fa` secondary, `#e9ecef` borders.
- **Surfaces (dark)**: `#0d0d0d` base, `#111111` panels, `#1a1a1a` cards, `#333333` borders.

### Typography
**Font: [Inter](https://fonts.google.com/specimen/Inter)** — the industry-standard for EdTech SaaS. Rationale:
- Optimized for screens at all sizes; excellent legibility for dense UI.
- Wide weight range (300–900) supports clear typographic hierarchy.
- WCAG AA compliant at standard body sizes.
- Feels modern and focused without being cold.

**Type scale (recommended)**:
| Role            | Size     | Weight |
|-----------------|----------|--------|
| Page title      | 1.5rem   | 700    |
| Section heading | 1.125rem | 600    |
| Body            | 0.9375rem| 400    |
| Label / caption | 0.8125rem| 500    |
| Micro / badge   | 0.75rem  | 500    |

### Design Principles
1. **Clarity over decoration** — every element earns its place. Text is readable, hierarchy is unambiguous, and noise is eliminated.
2. **Energy through structure** — the orange accent and strong weight contrasts create dynamism without clutter. Motion is subtle and purposeful (never decorative).
3. **AI is a first-class citizen** — AI features (AI Tutor panel, AI-generated audio/video) should feel powerful and trustworthy, never gimmicky. Use the accent color to signal AI touch-points.
4. **Role-adaptive density** — student views are guided and spacious; teacher/admin views can be denser and more data-rich, but always with clean hierarchy.
5. **WCAG AA by default** — all text/background combinations must meet 4.5:1 contrast in both light and dark modes. Interactive elements have clearly visible focus states. Reduced-motion media queries respected.

### Accessibility Requirements
- Target: **WCAG AA**
- All interactive components must have visible focus rings (not just outline: none overrides).
- Minimum touch target: 44×44px.
- Color is never the *only* differentiator (icons/labels always accompany color-coded states).
- `prefers-reduced-motion` should suppress non-essential animations.
